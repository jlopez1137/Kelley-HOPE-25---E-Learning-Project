What's working today

The digital human can now listen and speak.

Ask a question about any part of the course — the answer comes from the course itself.

Speak instead of typing — the tutor understands your voice.

Hear the answer read aloud — everything runs locally on the DGX.

Performance

Answers got 2–3× faster.

Before
30–60 s
per answer
→
After
15–25 s
warm · ~18 s typical

We right-sized the model from llama3.1:70b down to llama3.1:8b and reworked the backend so the pipeline is built once, kept warm in GPU memory, and never blocks while generating. Answer quality holds up for course Q&A — the speed is the difference students actually feel.

The avatar layer

Lip sync: evaluated, documented, not yet possible on this hardware.

The leading open-source avatar tools were evaluated on the DGX Spark, and none are compatible with the GB10 architecture today — the findings are documented for future cohorts. The interface already has a slot reserved for the moment a compatible solution exists.

Where we are

A working tutor today.
A foundation for what's next.

Students can navigate the PE 2.0 course, ask questions by text or voice, and get spoken, context-aware answers — all running locally, with nothing sent to the cloud.
